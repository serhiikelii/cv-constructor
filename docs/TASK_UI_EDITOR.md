# Задача: Реализация Панели Редактора (Left Sidebar)

**Цель:** Создать интерфейс ввода данных, разделенный на логические секции. Формы должны обновлять глобальный стейт приложения (Zustand) в реальном времени.

**Компоненты:** `EditorLayout`, `PersonalForm`, `ExperienceForm`, `EducationForm`, `SkillsForm`.
**Инструменты:** React Hook Form, Zod (валидация), Tailwind CSS, Lucide Icons.

---

## План реализации

### Шаг 1: Скелет и Навигация (Accordions)
Пользователь не должен видеть все поля сразу.
1.  **Создание компонента `Accordion`:**
    - Заголовок секции (например, "Опыт работы") + Иконка.
    - При клике раскрывается содержимое.
    - Логика: Может быть открыта только одна секция одновременно (для экономии места) или несколько (на выбор).
2.  **Структура сайдбара:**
    - Ширина: фиксированная (например, `w-1/3` или `min-w-[400px]`) на десктопе.
    - Скроллбар: Если форма длинная, скроллится только сайдбар, а не вся страница.

### Шаг 2: Форма "Личные данные" (Static Fields)
Самая простая форма.
1.  **Поля:** Имя, Фамилия, Должность (Job Title), Email, Телефон, Город, Ссылка на LinkedIn.
2.  **Фото:** Компонент загрузки файла (`input type="file"`).
    - *Важно:* Конвертировать изображение в Base64 строку и сохранять в стейт, чтобы оно отобразилось в превью без загрузки на сервер (для MVP).
3.  **Связка:** `onChange` каждого поля вызывает метод `setPersonalDetails` в сторе.

### Шаг 3: Динамические формы (Complex Arrays)
Секции "Опыт работы" и "Образование".
1.  **Список элементов:**
    - Отображение добавленных мест работы в свернутом виде (Например: "Google | Senior Dev").
    - Кнопки управления: "Редактировать" (карандаш), "Удалить" (корзина), "Переместить вверх/вниз" (стрелки).
2.  **Форма добавления/редактирования:**
    - Открывается при клике на "Добавить опыт".
    - Поля: Компания, Должность, Дата начала, Дата окончания (чекбокс "По настоящее время"), Описание (Textarea).
    - Валидация: Дата окончания не может быть раньше даты начала.

### Шаг 4: Форма "Навыки" (Tags Input)
1.  **Интерфейс:**
    - Поле ввода + кнопка "Enter" или "+".
    - При добавлении навык появляется ниже в виде "чипса" (тега) с крестиком для удаления.
2.  **Логика:**
    - В стейте хранится как массив строк `string[]`.

### Шаг 5: UX Улучшения
1.  **Rich Text (Опционально):** Для поля "Описание работы" и "О себе" подключить простой редактор (жирный, курсив, список), чтобы пользователь мог форматировать текст.
2.  **Прогресс-бар:** Визуальная шкала "Заполнено на 40%", мотивирующая заполнить все поля.
