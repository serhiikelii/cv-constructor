# Реализация Шаблона "Корпоративный" (The Executive)

**Статус:** В ожидании  
**Приоритет:** Высокий (MVP Core)  
**Цель:** Создать идеальный, строгий шаблон формата А4, оптимизированный для печати и систем ATS.

---

## Этап 1: Настройка "Листа бумаги" (Layout & Container)
В этом этапе мы создаем физическую основу документа.

- [ ] **Создание главного контейнера (Wrapper):**
    - Создать `div`, который будет служить страницей.
    - Установить фиксированную ширину **210mm** (стандарт A4).
    - Установить минимальную высоту **297mm** (чтобы даже пустой лист выглядел как страница).
    - Для отображения на экране (Screen view) добавить тень (`box-shadow`), чтобы лист выделялся на фоне.
- [ ] **Настройка полей (Margins/Padding):**
    - Установить внутренние отступы (`padding`) контейнера ровно **2.5cm** (или `25mm`) со всех сторон. Это стандарт делового документооборота.
- [ ] **Цветовая схема:**
    - Фон контейнера: Строго `#FFFFFF` (White).
    - Основной цвет текста: `#000000` (для печати) или `#1F2937` (Gray-800 для мягкости на экране).
- [ ] **Сброс стилей:**
    - Убедиться, что внутри этого контейнера не действуют глобальные стили приложения, которые могут исказить документ.

## Этап 2: Типографика (Typography)
Шрифты — единственный инструмент дизайна в этом шаблоне.

- [ ] **Подключение шрифтов:**
    - Импортировать шрифт с засечками (Serif). Рекомендация: **Merriweather**, **Lora** или системный **Times New Roman**.
    - Импортировать шрифт без засечек (Sans-Serif) только для мета-информации (опционально), например **Open Sans**.
- [ ] **Настройка Body:**
    - Размер шрифта: `10pt` или `11pt` (эквивалент `0.875rem` - `0.9rem` в Tailwind).
    - Межстрочный интервал (`line-height`): `1.5` или `1.6`. Текст не должен слипаться.
- [ ] **Стили заголовков (H1 - Имя):**
    - Размер: `24pt` - `28pt`.
    - Начертание: `Bold` (700).
    - Выравнивание: По центру (классика) или по левому краю (современная классика).
- [ ] **Стили секций (H2 - Опыт, Образование):**
    - Размер: `12pt` - `14pt`.
    - Регистр: `UPPERCASE` (Все заглавные).
    - Декор: Добавить нижнюю границу (`border-bottom`) толщиной `1px` или `2px` черного цвета. Отступ под линией — `4px`.

## Этап 3: Шапка профиля (Header)
Верхняя часть документа.

- [ ] **Блок Имени и Должности:**
    - Расположить друг под другом.
    - Должность должна быть меньше имени, обычного начертания (Regular), шрифт можно использовать чуть светлее (серый) для экрана, но черный для печати.
- [ ] **Блок Контактов:**
    - Вывод в одну строку (Flexbox Row).
    - Центрирование (если имя по центру).
    - **Разделители:** Вставить `span` с символом `|` или `•` между email, телефоном и ссылками. Добавить отступы вокруг разделителей.
- [ ] **Логика Фотографии:**
    - Создать условный рендеринг:
        - *Если фото нет:* Текст заголовка занимает 100% ширины.
        - *Если фото есть:* Контейнер становится Flex-row. Текст слева, фото справа (размер фото ~3.5cm x 4.5cm).

## Этап 4: Секция "Опыт работы" (Experience)
Самая сложная часть верстки. Используем Flexbox для разделения левой и правой части строки.

- [ ] **Контейнер элемента списка:**
    - Каждый опыт работы — это отдельный `article` или `div`.
    - Отступ снизу между работами (`margin-bottom`): `4mm`.
- [ ] **Строка 1: Должность и Даты:**
    - Использовать `Flexbox` -> `justify-content: space-between`.
    - Слева: **Название должности** (`font-weight: bold`).
    - Справа: **Период работы** (Например, "Янв 2020 — Наст. время"). Текст выровнен вправо.
    - *Важно:* Текст дат не должен переноситься на новую строку, если должность слишком длинная (`white-space: nowrap` для дат).
- [ ] **Строка 2: Компания и Локация:**
    - Аналогично `justify-content: space-between`.
    - Слева: **Название компании** (курсив `italic` или обычный).
    - Справа: **Город, Страна**.
- [ ] **Описание (Bullets):**
    - Тег `<ul>`.
    - Стиль маркера: `list-style-type: disc`.
    - Положение маркера: `list-style-position: outside` (чтобы текст второй строки не залезал под буллит).
    - Левый отступ (`padding-left`): `1.5em`.

## Этап 5: Оптимизация для ATS (Semantic & Code Structure)
Техническая чистота кода для роботов.

- [ ] **Семантика:**
    - Использовать теги `<section>` для разделов (Опыт, Образование).
    - Использовать `<h2>` для заголовков разделов.
    - Использовать `<h3>` для названий должностей/университетов.
- [ ] **DOM Порядок:**
    - Убедиться, что в коде блоки идут в логическом порядке чтения, даже если CSS меняет их визуально (хотя в Классике CSS порядок обычно совпадает с HTML).
- [ ] **Запрет таблиц:**
    - Строгое правило: не использовать CSS `display: table` или HTML `<table>` для лейаута. Только Flexbox.

## Этап 6: Пустые состояния (Empty States)
Логика чистоты документа.

- [ ] **Условный рендеринг секций:**
    - Проверка: Если массив `skills` пуст -> не рендерить заголовок "НАВЫКИ" и разделительную линию.
    - Проверка: Если поле `summary` (О себе) пустое -> не рендерить этот блок.
- [ ] **Баланс белого:**
    - Если контента мало (резюме занимает полстраницы), динамически увеличить отступы (`gap`) между основными секциями, чтобы визуально заполнить лист (опциональная фича).

## Этап 7: Стили для печати (Print CSS)
Критический этап для экспорта в PDF.

- [ ] **Media Query:**
    - Весь код ниже оборачивается в `@media print { ... }`.
- [ ] **Управление разрывами (Page Breaks):**
    - Для каждого блока "Место работы" (`article`) задать CSS: `break-inside: avoid`. Это запретит принтеру разрывать описание одной работы на две страницы.
    - Для заголовков секций (`h2`) задать: `break-after: avoid` (заголовок не должен остаться внизу страницы один, без контента).
- [ ] **Очистка стилей:**
    - Скрыть все элементы UI (кнопки "Скачать", меню, границы редактора). `display: none`.
    - Убрать тени контейнера (`box-shadow: none`).
    - Установить `margin: 0` для `body`, чтобы браузер не добавлял свои поля к нашим 2.5см.
- [ ] **Ссылки:**
    - `text-decoration: none` (убрать подчеркивание).
    - `color: #000` (цвет текста).
    - Ссылки должны оставаться рабочими (кликабельными) в PDF.
